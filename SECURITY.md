# Security Policy

## Supported versions

Only the latest release on the `master` branch receives security fixes.

| Version | Supported |
|---|---|
| 2.x (current) | Yes |
| 1.x | No |

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

### Option 1 — GitHub private advisory (preferred)

Use GitHub's private vulnerability reporting:

[https://github.com/nixrajput/portfolio-nextjs/security/advisories/new](https://github.com/nixrajput/portfolio-nextjs/security/advisories/new)

### Option 2 — Email

Send a description of the vulnerability to **nikhil@starapps.studio**. Include:

- A clear description of the issue and its potential impact.
- Steps to reproduce or a proof-of-concept.
- Any suggested remediation if you have one.

## Response expectations

| Milestone | Target |
|---|---|
| Acknowledgement | Within 72 hours |
| Initial assessment | Within 7 days |
| Fix or mitigation | Within 30 days for high-severity issues |

You will be credited in the release notes unless you prefer to remain anonymous. Thank you for helping keep this project secure.
